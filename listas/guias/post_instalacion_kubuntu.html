Guía de configuración post-instalacion de Kubuntu

Introducción:

Una vez que hayamos terminado de instalar el sistema operativo kubuntu, necesitaremos configurar el sistema, para que éste tenga todo lo necesario para que cualquier empleado municipal pueda utilizarlo. El sistema operativo, el cual esta basado en la distribución de linux Ubuntu, cuenta con un archivo, sources.list, el cual se encuentra en /etc/apt/sources.list en el cual se encuentran los direccionamientos a los repositorios a los cuales el SO acude cuando necesita actualizarse o descargar alguna aplicación.

Actualizar sistema

Para actualizar nuestra sistema deberemos abrir la consola, y tipear los siguientes comandos:

Descargar actualizaciones:

sudo apt-get update

Con este comando descargaremos las actualizaciones del sistema, las cuales se descargaran de los repositorios mencionados anteriormente.

En caso que salga un error que no tengamos permisos de usuario, deberemos hacerlo como usuario administrador, esto lo haremos con el siguiente comando. 

su nombre_usuario_admin

Generalmente, el nombre de usuario administrador es cil, en caso de que no exista, podemos entrar como usuario root, o sea, el nombre de usuario será root. Una vez hayamos entrado como usuario administrador, ahora si podremos tipear el comando mencionado anteriormente.

Una vez descargadas las actualizaciones, deberemos instalarlas. Lo hacemos con el siguiente comando:

Instalar actualizaciones:

sudo apt-get update

Una vez hayamos hecho esto, tendremos nuestro sistema actualizado.

Error común

Puede pasar que al intentar hacer los pasos anterior para actualizar el sistema, al momento de descargar las actualizaciones nos salga un error, de que no se encontraron los repositorios a los cuales apunta nuestro archivo sources.list, el cual mencionamos al principio. Para hacer esto, deberemos modificar este archivo. Nuevamente, abrimos la consola y nos dirijimos a la siguiente ubicacion:

Moverse al directorio del archivo:

sudo cd /etc/apt

Nos crearemos un archivo de backup de nuestro archivo sources.list, en caso de que hagamos algo mal.

Crear backup:

sudo cp sources.list sources_backup.list

Antes de modificar el archivo, necesitamos saber la versión de nuestro sistema operativo, lo haremos con el siguiente comando.

Consultar versión del SO:

lsb_release -a

Generalmente, tendremos la versión 18.04. En caso que tengamos esta versión, la siguiente plantilla funciona para nuestro archivo sources.list.

Plantilla de sources.list kubuntu 18.04

En caso que no tengamos esta versión, deberemos buscar una plantilla para la versión correspondiente, tendremos que googlear. Generalmente aparece como sources.list ubuntu (version del sistema) plantilla.

Para modificar el archivo sources.list, debemos tipear los siguientes comandos

Abrir archivo con editor de texto:

sudo nano /apt/etc/sources.list

Borramos todo lo que tenga el archivo, y pegamos nuestra plantilla. Luego guardamos el archivo.

Ahora, con esto, se deberia arreglar el error que teníamos, y podremos ahora si, seguir los pasos que hicimos anteriormente y actualizar el sistema.
